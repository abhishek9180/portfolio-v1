# Phase 4 — SEO & Metadata

## Objective

Replace all placeholder SEO configuration with metadata that accurately represents the portfolio and follows modern SEO best practices.

The goal is to ensure the portfolio is fully optimized for:

- Search engines
- Social media sharing
- Rich previews
- Portfolio discoverability
- Personal branding

---

## Required Context

Before implementation, read:

- `AGENTS.md`
- `specs/assets/profile.md`
- `specs/assets/content-style.md`

---

## Scope

Perform a complete SEO audit of the project and update every applicable configuration.

---

# 1. Global Metadata

Update all global metadata.

Including (but not limited to):

- Site title
- Default page title
- Title template
- Description
- Author
- Keywords
- Category
- Application name
- Generator (if applicable)

Ensure every value reflects the portfolio owner.

---

# 2. Metadata Per Page

Review every page.

Ensure each page has an appropriate:

- title
- description
- canonical URL
- Open Graph title
- Open Graph description

Pages should not reuse generic metadata unless appropriate.

---

# 3. Open Graph

Update all Open Graph configuration.

Verify:

- Site name
- Title
- Description
- URL
- Preview image
- Image dimensions
- Image type
- Locale

Replace placeholder images if necessary.

---

# 4. Twitter / X Cards

Configure Twitter metadata.

Including:

- Card type
- Title
- Description
- Preview image
- Twitter username (if available)

---

# 5. Favicons & App Icons

Audit every application icon.

Including:

- favicon.ico
- apple-touch-icon
- android icons
- mask icon
- web app manifest icons

Replace placeholder branding where required.

---

# 6. Web Manifest

Review the web app manifest.

Update:

- name
- short_name
- description
- theme_color
- background_color
- icons

Ensure branding is consistent.

---

# 7. Robots

Review robots configuration.

Verify:

- robots.txt
- metadata robots settings
- indexing configuration

The production portfolio should be indexable.

---

# 8. Sitemap

Review sitemap generation.

Verify:

- homepage
- work pages
- project pages
- blog pages (if enabled)

Every public page should appear.

---

# 9. Canonical URLs

Ensure canonical URLs exist where appropriate.

Avoid duplicate URL indexing.

---

# 10. Structured Data

Audit JSON-LD or structured metadata.

Implement or update where supported.

Prefer:

- Person
- WebSite
- Breadcrumb (if applicable)

Use information from profile.md.

Do not invent data.

---

# 11. Social Preview Images

Verify social preview images.

Ensure:

- correct dimensions
- high quality
- appropriate branding

Replace placeholder images if necessary.

---

# 12. Browser Metadata

Review:

- theme color
- browser color
- viewport metadata
- application metadata

Ensure consistency.

---

# 13. URL Configuration

Verify all generated URLs use the production domain.

Including:

- canonical URLs
- OG URLs
- sitemap
- RSS (if available)

No localhost or placeholder domains should remain.

---

# 14. Metadata Consistency

Ensure metadata is consistent across:

- homepage
- work pages
- project pages
- blog pages

Avoid conflicting titles and descriptions.

---

# 15. Accessibility Metadata

Verify:

- language
- charset
- viewport

Remain unchanged unless improvements are required.

---

# Out of Scope

Do not modify:

- Portfolio content
- Work experience
- Projects
- Styling
- Layout
- Components
- Routing

unless required to support SEO.

---

# Deliverables

- All placeholder SEO removed
- Metadata updated
- Open Graph configured
- Twitter cards configured
- Favicons reviewed
- Manifest updated
- Sitemap verified
- Robots verified
- Structured data verified
- Social preview images configured
- Canonical URLs configured
- No placeholder domains remain

---

# Acceptance Criteria

- Every public page has appropriate metadata.
- Open Graph previews display correctly.
- Twitter cards display correctly.
- Sitemap includes all public pages.
- Robots configuration allows indexing.
- Structured data validates successfully.
- No placeholder titles remain.
- No placeholder descriptions remain.
- No placeholder domains remain.
- Build passes.
- TypeScript passes.
- Lint passes.

---

# Implementation Notes

The agent should not assume the template's existing SEO configuration is complete.

Instead, perform a repository-wide audit to identify every SEO-related file and configuration before making changes.

This includes (but is not limited to):

- metadata configuration
- layout files
- page metadata
- manifest
- robots
- sitemap
- Open Graph images
- favicons
- structured data
- environment variables
- SEO utility functions

Document any template limitations in `docs/decisions.md`.
